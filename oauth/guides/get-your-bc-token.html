Descripción general

NOTA: El método preferido para obtener las credenciales del cliente es utilizar la interfaz de usuario de administrador de Studio. Este documento es para quienes eligen utilizar la API de OAuth en su lugar.

El mejor y primer paso para comprender la información de este documento es mirar la Descripción general de OAuth si aún no lo ha hecho.

Tu BC_TOKEN es un identificador único y anónimo para usted como usuario de Brightcove. Además de su existencia en un sistema backend de Brightcove, se configura como una cookie cuando inicia sesión en Studio para que Studio conozca su función y las cuentas a las que tiene acceso. Todo esto es transparente y no requiere ninguna acción por su parte.

El uso de la BC_TOKEN para usted está en la autenticación de una solicitud de API de OAuth para crear credenciales de cliente.

Notas:
  • La BC_TOKEN El valor no tiene relación con los tokens de API Management con los que trabaja en Studio.
  • Si usted utilizar el administrador de Studio para crear credenciales de cliente , no es necesario recuperar su BC_TOKEN - Studio se encargará de eso por ti.
  • los BC_TOKEN cookie es una cookie de sesión y caduca cada vez que cierra sesión en Studio (también caduca automáticamente después de 24 horas). Cada vez que un nuevo BC_TOKEN La cookie se elimina cuando inicia sesión en Studio, tendrá un nuevo valor.

Requisitos

  • Una cuenta de Brightcove válida con permisos de administrador necesarios para crear credenciales de cliente
