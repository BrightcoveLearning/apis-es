Introducción

Al ingerir videos, puede usar archivos de manifiesto XML para especificar metadatos adicionales (incluido el título del video, la descripción, los tiempos de vuelo, las etiquetas y los metadatos personalizados), así como los archivos de miniaturas y subtítulos opcionales.

Nota:
  • Esto es un beta servicio de ingestión: comuníquese con su administrador de cuentas si está interesado en participar en la versión beta.
  • Aunque las etiquetas XML utilizan la palabra clave ooyala , este método de ingestión le permitirá cargar videos a su cuenta de Video Cloud.

Manejo de metadatos

Si incluye metadata para los videos cuando los ingiere, se asigna a los campos de video de Video Cloud de una de estas dos formas:

  1. Por defecto , cada metadata El par clave / valor se agrega como un etiqueta en la forma: meta::key::value. Por ejemplo, <ooyala:metadata name="category">cooking</ooyala:metadata> produciría la etiqueta meta::category::cooking

Elementos del manifiesto XML

Puede utilizar los siguientes elementos en el archivo de manifiesto XML.

Elementos XML
Etiqueta XML Descripción Requerido / Opcional
<channel> Contiene uno o más <item> elementos. requerido
<item> Representa un solo video para ingerir. Puede especificar varios videos (<item> etiquetas) en un archivo de manifiesto XML. Contiene las etiquetas restantes en esta tabla. requerido
<media:content> Especifica un video para ingerir como el Nombre del archivo de un video que subiste a tu servidor.
Notas:
  • La extensión del archivo URL remoto debe ser m3u8 o mpd ; de lo contrario, lo estableceremos en progresivo .mp4.
  • Para un activo remoto, use <ooyala:remoteasset> en lugar de.
 requerido (excepto para activos remotos)
<media:title> Nombre o título del video. opcional
<media:thumbnail> Especifica una miniatura (imagen de vista previa). Uno de los siguientes:
  • Nombre de archivo de una miniatura que cargó en su servidor.
  • URL donde se encuentra la miniatura. La URL puede incluir http:// o https://.
 opcional
<media:subTitle> Especifica un archivo de subtítulos cerrados (compatible con archivos DFXP, WebVTT y SRT). Ver Ingesta de archivos de subtítulos ocultos. Atributos:
  • href(requerido ) - Uno de los siguientes:
    • Nombre de archivo del archivo de subtítulos que cargó. Ejemplo: subtítulo-1.dfxp
    • URL donde se encuentra el archivo de subtítulos. La URL puede incluir http:// o https://.
  • lang(requerido ): idioma del archivo de subtítulos. Código de dos letras. Ver Subtítulos opcionales admitidos.
  • frameRate(Opcional ) - Velocidad de fotogramas del archivo de subtítulos cerrados. Las unidades están en cuadros por segundo (FPS). El valor predeterminado es 30. Los archivos de subtítulos cerrados más antiguos pueden tener 24.
 opcional
<media:description> Descripción del video. opcional
<> URL permanente donde incrusta el video. Se asigna a los activos creados hosted_at propiedad opcional
<dcterms:valid> Horas de vuelo que representan las horas de inicio y finalización en las que se puede reproducir el activo. Ejemplo: 
<dcterms: valid> start = 2011-06-28T15: 15: 00-07: 00; end = 2022-12-01T16: 00: 00-07: 00; esquema = W3C-DTF </ dcterms: valid>
 opcional
<ooyala:labels> Uno o más valores, separados por comas. Ejemplo: /sports,/sports/jogging. Estos valores se utilizarán para completar Etiquetas de Video Cloud. opcional
<ooyala:embedcode> Identificación de video de Video Cloud. opcional (obligatorio si está actualizando un video en lugar de agregar uno nuevo)
<ooyala:metadata> Metadatos personalizados para el video. Se asigna a campos personalizados en Studio.
Nota: Si se definen metadatos y no existe un campo personalizado, la ingestión fallará.

Ejemplo: <ooyala:metadata name="category">cooking</ooyala:metadata>.

 opcional
<ooyala:remoteasset> Especifica un activo remoto para ingerir. Un activo remoto es un video alojado en otro lugar que no sea Video Cloud. Especifique los siguientes atributos:
  • durationInMs: duración del medio (en milisegundos)
  • url: URL donde se encuentra el activo remoto. La URL puede incluir http:// o https://.

Esta configuración se asigna al valor de la propiedad hospedada_at para un activo remoto. Ejemplo: <ooyala:remoteasset durationInMs='5000' url="http://your_website/your_remote_assets.mp4" />

Nota: Una vez que se ha creado un activo remoto, su propagación a las distintas CDN puede demorarse 60 segundos o más. Si solicita un activo remoto demasiado pronto después de su creación, los CDN almacenarán en caché los resultados, lo que puede tardar varios minutos en borrarse.
 requerido (solo para activos remotos)
<ooyala:profileguid> Especifica un perfil de ingesta de Video Cloud que se utilizará para procesar el vídeo. Si no desea usar el perfil predeterminado de la cuenta, debe especificar el que desea usar en su lugar. Ver Gestión de perfiles de ingesta. opcional

Paso a paso

Creación de un archivo de manifiesto XML

  1. Sube tus videos y otro contenido (miniaturas, archivos de subtítulos, etc.) a tu servidor de alojamiento. Las URL deben estar disponibles públicamente.
  2. Abra un editor de texto, preferiblemente uno que valide XML. (El equipo de Brightcove Learning Services utiliza Código de Visual Studio pero hay varias otras buenas opciones disponibles).
  3. Agregue los encabezados XML necesarios. Por ejemplo: 
    <? xml version = "1.0" encoding = "utf-8"?> <rss version = "2.0" xmlns: media = "http://search.yahoo.com/mrss/" xmlns: dcterms = "http: / /purl.org/dc/terms/ "xmlns: fh =" http://purl.org/syndication/history/1.0 "xmlns: ooyala =" http://www.brightcove.com/mrss/ ">
  4. Crear un <channel> envase.
  5. Crear un <item> entrada para cada video, que contiene los elementos deseados de la tabla anterior.

    En el siguiente ejemplo, el XML especifica metadatos para dos videos. Para el primer video, el archivo y las miniaturas se encuentran en un servidor de carga y se extraen de ese servidor. Para el segundo video, el archivo y la miniatura ya se cargaron en un servidor Ooyala. El segundo video también tiene horarios de vuelo.

    <? xml version = "1.0" encoding = "utf-8"?> <rss version = "2.0" xmlns: media = "http://search.yahoo.com/mrss/" xmlns: dcterms = "http: / /purl.org/dc/terms/ "xmlns: fh =" http://purl.org/syndication/history/1.0 "xmlns: ooyala =" http://brightcove.com/mrss "> <canal> <elemento > <media: content url = "http://mysite.com/upload/lacrosse_70.mov" /> <media: thumbnail url = "http://mysite.com/upload/lacrosse_70_previewimage.jpg" /> <media: title> Deportes que realmente existen: Lacrosse</media:title > <Media:Description>Mi descripción de mi vídeo</media:description > <ooyala:etiquetas>/sports/lacrosse,/ hobbies/lacrosse</ooyala:labels > <link> http://mysite.com/videos/sports/real_sports.html</link > <ooyala:metadata name=» video_id">70</ooyala:metadatos > <ooyala:metadata name="season_number">1</ooyala:metadata > </item > <item> <media:content url=» curling_71.mov "/> <media:miniatura url=» curling_71_previewimage.jpg "/> <Media:título>Deportes que realmente existen: Curling </ media: title> <media: description> Mi descripción de mi video </ media: description> <ooyala: labels> / sports / curling, / hobbies / curling </ ooyala: labels> <link> http: // mysite.com/videos/sports/real_sports.html </link> <ooyala: metadata name = "video_ID"> 71 </ ooyala: metadata> <ooyala: metadata name = "season_number"> 1 </ ooyala: metadata> < ! - establecer tiempos de vuelo -> <dcterms: valid> start = 2011-06-28T15: 15: 00-07: 00; end = 2022-12-01T16: 00: 00-07: 00; esquema = W3C-DTF </ dcterms: valid> </item> </channel> </rss>
  6. Guarde el archivo como XML. Asegúrese de especificar un nombre exclusivo. Si especifica dos archivos con el mismo nombre y carga uno mientras se procesa el otro, es posible que se ignore el segundo.
  7. Con el método de carga que elija, cargue el archivo de manifiesto XML en su servidor de alojamiento donde cargó los videos.
    Recuerde que la URL del manifiesto XML debe coincidir exactamente con la que le proporcionó a su administrador de cuentas.
