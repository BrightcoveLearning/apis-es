Campos primarios

Tabla 1: Campos primarios
Campo Tipo Solo lectura Defecto Descripción
id cuerda sí ninguno la identificación de la lista de reproducción
account_id cuerda sí ninguno el id de la cuenta
created_at Cadena de fecha y hora ISO 8601 sí ninguno la lista de reproducción de fecha y hora se agregó a la cuenta; ejemplo: "2014-12-09T06: 07: 11.877Z"
description cuerda no nulo la breve descripción de la lista de reproducción: 248 caracteres de un solo byte como máximo
favorite booleano no falso si la lista de reproducción está incluida en favoritos
limit número no 100 número de videos que puede contener la lista de reproducción (número entero, 1 - 100) devuelto solo para listas de reproducción inteligentes
name cuerda no ninguno Título de la lista de reproducción: campo obligatorio (longitud máxima: 248 caracteres de un solo byte)
reference_id cuerda no nulo cualquier valor que sea único dentro de la cuenta
search cuerda no nulo cadena de búsqueda utilizada para definir los videos de una lista de reproducción inteligente (ejemplo: "+ etiquetas:" pájaro "," pájaros "")
type cuerda sí el tipo de lista de reproducción Ver nota[1] debajo
updated_at Cadena de fecha y hora ISO 8601 sí ninguno la lista de reproducción de fecha y hora se modificó por última vez;[1-2] ejemplo: "2015-01-13T17: 45: 21.977Z"
video_ids formación no variedad de identificadores para los videos en la lista de reproducción Devuelto solo para listas de reproducción manuales

Notas

  • [1]

    La type El valor solo puede ser uno de: EXPLÍCITO, ACTIVATED_OLDEST_TO_NEWEST , ACTIVATED_NEWEST_TO_OLDEST , ALPHABETICAL , PLAYS_TOTAL , PLAYS_TRAILING_WEEK , START_DATE_OLDEST_TO_NEWEST,START_DATE_NEWEST_TO_OLDEST

    EXPLICIT significa que es una lista de reproducción manual; todos los demás tipos son listas de reproducción inteligentes con nombres que corresponden a su esquema de ordenación.
