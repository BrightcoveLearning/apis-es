Descripción general

Esta muestra y el código asociado se proporcionan como una guía para su desarrollo de producción. El código no está certificado para uso en producción ni se admite oficialmente.
Tenga en cuenta que esta aplicación se creó para ayudar a los clientes de Brightcove a migrar del antiguo Smart Player al nuevo Brightcove Player. Se creó en torno a videos agregados mediante el sistema de ingesta heredado y no devuelve resultados significativos para los videos de Dynamic Delivery.

Esta herramienta genera un informe que incluye lo siguiente:

  • Resumen que muestra cuántos videos tienen No Reproducciones HLS o MP4 o ambas
  • Informe de cada video que muestra las representaciones HLS y MP4 (HTTP) (con el rango de velocidad de bits y el tamaño de fotograma para cada uno)

El objetivo es ayudarlo a prepararse para migrar de Smart Player a Brightcove Player identificando videos que deben transcodificarse para crear versiones HLS y / o MP4.

Tenga en cuenta que los videos de Dynamic Delivery no están marcados, ya que no es necesario.

Obtener credenciales

Como la mayoría de las API RESTful de Brightcove, la API de CMS requiere tokens de acceso de OAuth para autorizar solicitudes. Para obtener tokens de acceso, primero debe obtener credenciales de cliente con permisos para los tipos de operaciones que desea solicitar.

  1. Para obtener sus credenciales, siga las instrucciones en Administrar las credenciales de la API. Esta es la forma recomendada de obtener sus credenciales de cliente. Esta muestra solo requiere Video Read permisos, pero en la mayoría de los casos, es probable que desee obtener todos los permisos:

    También puede obtener credenciales mediante la API de OAuth, siguiendo las instrucciones de una de las guías que se enumeran a continuación:

  • OAuth: Obtener credenciales de cliente mediante CURL
  • OAuth: Obtenga las credenciales del cliente con Postman

Inspector de entrega

Puede usar la aplicación a continuación para generar un informe sobre las representaciones de todos los videos en su cuenta. Los videos sin versiones HLS o HTTP MP4 (descarga progresiva) deberán volver a codificarse para que funcionen bien en el nuevo reproductor. Los videos que solo tengan representaciones HLS de baja tasa de bits también deben volver a codificarse.

Ver la pluma Inspector de reproducciones de video por Brightcove Learning Services ( rcrooks1969 ) en CodePen.

Código de proxy

Para crear su propia versión de la aplicación de muestra en esta página, debe crear y alojar su propio proxy. (Los proxies utilizados por Brightcove Learning Services solo aceptan solicitudes de dominios de Brightcove). Un proxy de muestra, muy similar al que usamos, pero sin las comprobaciones que bloquean las solicitudes de dominios que no son de Brightcove, se puede encontrar en este repositorio de GitHub. También encontrará instrucciones básicas para usarlo allí, y una guía más elaborada para crear aplicaciones alrededor del proxy en Usando las API REST.
