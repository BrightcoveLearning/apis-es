Actualizaciones automáticas de jugadores

De forma predeterminada, Brightcove actualiza automáticamente todos los reproductores de forma regular para que puedan aprovechar las nuevas funciones y correcciones sin ningún esfuerzo por parte de nuestros clientes. Si bien Brightcove no recomienda optar por no recibir actualizaciones en la mayoría de los casos, Brightcove se da cuenta de que, de vez en cuando, algunos clientes prefieren que sus reproductores permanezcan completamente intactos (por ejemplo, ha realizado pruebas rigurosas en preparación para un gran evento y no quiere arriesgarse a que algo cambie).

Optar por no recibir actualizaciones automáticas

Si encuentra la necesidad de optar por no recibir actualizaciones del reproductor durante un período de tiempo, la forma más sencilla es aplicando un PATCH a la configuración de su reproductor. La siguiente declaración curl realiza esta tarea:

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request PATCH \
    --data '{
        "player": {
            "template": {
              "locked": true
            }
        }
    }' \
    https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration

Esto agrega la configuración bloqueada a su reproductor de vista previa. El PATCH afecta inicialmente solo a su reproductor de vista previa, por lo que debe publicar su reproductor. Puede hacer esto en Studio o usando la siguiente declaración curl:

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request POST \
    https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/publish

Optar por volver a las actualizaciones automáticas

Cuando esté listo para volver a suscribirse a las actualizaciones, simplemente aplique otra actualización, configurando locked a false y luego vuelva a publicar el reproductor. Primero cambia bloqueado a falso:

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request PATCH \
    --data '{
      "player": {
        "template": {
          "locked": false
        }
      }
        }' \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration

Esto actualizará su reproductor de vista previa a la versión actual de la plantilla del reproductor si es diferente de la que tiene actualmente. Debe probar con la URL del reproductor de vista previa que devuelve esta llamada para asegurarse de que su reproductor aún funcione como se espera. Una vez satisfecho, publique el reproductor para que se active el nuevo reproductor:

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request POST \
    https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/publish

Cambiar a una versión específica

Es posible que desee cambiar su reproductor a una versión específica. Es posible que esté optando por no recibir actualizaciones, pero desea usar alguna versión específica que vino después de optar por no recibir actualizaciones. Para hacer esto: use la siguiente declaración curl, por supuesto, sustituyendo la versión específica del reproductor que desea:

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request PATCH \
    --data '{
      "player": {
        "template": {
          "version": "1.14.11"
        }
      }
        }' \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration

Tenga en cuenta que puede configurar la versión y optar por no recibir actualizaciones al mismo tiempo:

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request PATCH \
    --data '{
      "player": {
        "template": {
          "version": "1.14.11",
          "locked": true
        }
      }
        }' \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration

Al igual que con cualquier actualización de PATCH, debe publicar el reproductor una vez que esté satisfecho con los cambios:

    curl \
    --header "Content-Type: application/json"cp
    --user $EMAIL \
    --request POST \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/publish
