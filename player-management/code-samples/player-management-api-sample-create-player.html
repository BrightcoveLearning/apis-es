Descargo de responsabilidad

Esta aplicación de muestra está pensada como una forma de explorar los conceptos básicos de la API de administración de reproductores. No está destinada a ser una aplicación implementable lista para la empresa. Por ejemplo, la validación de formularios no es exhaustiva y las contraseñas NO se envían utilizando HTTPS.

Para conocer el enfoque general sobre la arquitectura de esta aplicación de muestra, específicamente la autenticación y las implementaciones de AJAX, consulte la ejemplos que usan HTTPRequest.

Arquitectura de la aplicación

La lógica de la aplicación de alto nivel para esta aplicación es la siguiente:

  1. Reúna los datos de usuario necesarios para crear un reproductor.
  2. Construya la estructura de datos JSON para pasar a la API para la creación de reproductores
  3. Al hacer clic en el botón, realice una llamada AJAX para crear un reproductor
  4. En el éxito de la llamada AJAX, realice lo siguiente:
    • Inyectar datos de respuesta JSON en una página HTML
    • Función de llamada para publicar reproductor
  5. Realice una llamada AJAX para publicar un reproductor
  6. En el éxito de la llamada AJAX, realice lo siguiente:
    • Inyectar datos de respuesta JSON en HTML
    • Cree HTML dinámicamente que muestre la información del reproductor seleccionado e inyéctelo en la página HTML

Código de proxy

Para crear su propia versión de la aplicación de muestra en esta página, debe crear y alojar su propio proxy. (Los proxies utilizados por Brightcove Learning Services solo aceptan solicitudes de dominios de Brightcove). Un proxy de muestra, muy similar al que usamos, pero sin las comprobaciones que bloquean las solicitudes de dominios que no son de Brightcove, se puede encontrar en este repositorio de GitHub. También encontrará instrucciones básicas para usarlo allí, y una guía más elaborada para crear aplicaciones alrededor del proxy en Usando las API REST.
