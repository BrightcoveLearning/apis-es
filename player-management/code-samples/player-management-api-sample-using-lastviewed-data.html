Descargo de responsabilidad

Esta aplicación de muestra está pensada como una forma de explorar los conceptos básicos de la API de administración de reproductores. No está destinada a ser una aplicación implementable lista para la empresa. Por ejemplo, la validación de formularios no es exhaustiva y las contraseñas NO se envían utilizando HTTPS.

Para conocer el enfoque general sobre la arquitectura de esta aplicación de muestra, específicamente la autenticación y las implementaciones de AJAX, consulte la ejemplos que usan HTTPRequest.

Comprender la información last_viewed

Cada reproductor tiene asociada una matriz que contiene un registro de cuántas veces se cargó un reproductor. Esta información se almacena en un nombre de matriz last_viewed. A continuación, verá un ejemplo de una matriz de este tipo. Tenga en cuenta que el count registra cuántas veces en el día se cargó el reproductor específico. La date especifica el día específico.

    [
      {
        "count": 13,
        "date": "2018-10-24T05:00:00.000Z"
      }, {
        "count": 22,
        "date": "2018-10-23T05:00:00.000Z"
      }, {
        "count": 40,
        "date": "2018-10-22T05:00:00.000Z"
      }, {
        "count": 2,
        "date": "2018-10-21T05:00:00.000Z"
      }
    ]
Tenga en cuenta lo siguiente:
  • El recuento se compila una vez al día.
  • El recuento es para cargas de reproductores, no es necesario que se reproduzca un video para agregarlo al recuento.
  • La matriz está limitada a 30 entradas. Lo más probable es que NO sean 30 días consecutivos, sino 30 días en los que el recuento no fue cero.
  • Los datos compilados NO registran recuentos antes de mediados de octubre de 2018.
  • Si un reproductor no tiene datos, la matriz está vacía.

Arquitectura de la aplicación

La lógica de la aplicación de alto nivel para esta aplicación es la siguiente:

  1. Reúna los datos de usuario necesarios para recuperar la información de todos los reproductores en una cuenta
  2. Construya la estructura de datos JSON para pasar a la API para la creación de reproductores
  3. Al hacer clic en el botón, realice una llamada AJAX para recuperar información de todos los reproductores en una cuenta
  4. En caso de éxito de la llamada AJAX, extraiga el last_viewed matriz para cada reproductor.
  5. Para cada reproductor, muestre el last_viewed información en la tabla
