Introducción

La información de este tema se aplica principalmente a los perfiles de ingesta personalizados. Perfiles estándar todos los maestros de archivo (ver debajo)

Si archiva un archivo maestro cuando ingiere un video, podrá volver a transcodificar el video más tarde sin volver a ingerirlo. Este tema explica sus opciones para archivar sus masters. Tenga en cuenta que archivar el maestro significa almacenar el archivo de video maestro en su cuenta para que esté disponible para su procesamiento futuro. Ver Creación de perfiles de ingesta para una entrega dinámica para más información.

Tenga en cuenta que si almacena el maestro digital cuando ingiere el video y luego vuelve a codificar el video usando un perfil que no archiva los maestros, el maestro digital para el video no eliminarse y aún podrá volver a codificar el vídeo.

Archivar un maestro

Para archivar un master para su video, debe incluir el digital_master objeto en su perfil de ingesta:

    {
        "account_id": 3423487123048,
        "digital_master": {
            "rendition": "passthrough",
            "distribute": false
        },
        "name": "nondrm-default",
        ...
    }

Fuente o interpretación

Para los perfiles de Entrega dinámica (incluida la codificación sensible al contexto), puede archivar el archivo de origen de video como maestro (en otras palabras, configurar "rendition": "passthrough" en el digital_master parte del perfil). El digital_master no ser distribuido (enviado a la CDN como una versión disponible) para estos perfiles, incluso si establece "distribute": true.

Para los perfiles de Legacy Ingest, puede optar por utilizar su archivo de origen de vídeo como maestro, o puede seleccionar una reproducción de alta calidad para usar como maestro.

Para usar el archivo fuente de video que ingiere como maestro, incluya el siguiente código en su perfil de ingesta:

    "digital_master": {
        "rendition": "passthrough",
        "distribute": false
    }

Para usar una interpretación para el maestro en lugar de su archivo de origen (solo perfiles de ingesta heredados), configure el rendition campo de la digital_master objetar a la Identificación de referencia para la interpretación:

    "digital_master": {
            "rendition": "av0",
            "distribute": true
        },
        "name": "nondrm-av0",
        "packages": [],
        "renditions": [
        {
            "media_type": "video",
            "reference_id": "av0", ...

Si pones rendition a none , no se archivará ningún maestro y lo harás no poder retranscodificar el video. Sin embargo, puedes reemplazar el video ingiriéndolo de nuevo.

Distribución

La distribución solo funcionará para Perfiles de ingesta heredada.

Puede elegir si el maestro se distribuye o no. Distribuido significa que el maestro se envía a la CDN y está disponible como una interpretación reproducible. Si el maestro no se distribuye, Video Cloud lo archiva pero no lo envía a la CDN.

Notas:
  • Si seleccionó una de sus interpretaciones como maestra y configuró distribute a false , esa interpretación no se enviará a la CDN y no estará disponible para su reproducción
  • Si la interpretación se establece en passthrough y distribute a true , el master solo se distribuirá si está en formato h264
  • El video original puede modificarse para que se pueda reproducir en Internet. Sin embargo, el archivo maestro archivado será idéntico al archivo original, excepto en casos muy raros en los que necesitemos modificarlo para transcodificarlo.
  • Distribuir el maestro hace no funciona para videos ingeridos usando Entrega dinámica perfiles. Si observa los perfiles de ingesta de Dynamic Delivery, verá que distribute se establece en false. Si duplica el perfil e intenta cambiarlo en Studio, el valor volverá a ser false automáticamente.

Para distribuir el master al CDN, configure "distribute": true en el digital_master objeto:

    "digital_master": {
        "distribute": true,
        "rendition": "passthrough"
    }

Para evitar que el maestro se distribuya, configure distribute a falso:

    "digital_master": {
        "distribute": false,
        "rendition": "passthrough"
    }

Comportamiento por defecto

Por defecto (si el perfil no incluye un digital_master campo), el API de ingesta dinámica y Módulo de carga se comportará exactamente como si se hubiera incluido lo siguiente en el perfil:

    "digital_master": {
        "distribute": false,
        "rendition": "none"
    }

En otras palabras, no se almacenará ningún maestro digital.

Si tiene lo siguiente en su perfil:

    "digital_master": {
        "distribute": false
    }

... sin rendition campo para especificar qué se debe archivar, el video de origen será archivado como el maestro digital.
