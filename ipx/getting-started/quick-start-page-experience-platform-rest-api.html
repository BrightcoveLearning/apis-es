Introducción

La API de experiencia en la página le permite administrar las experiencias en la página de manera programática, lo que la hace particularmente útil para las integraciones de CMS, para permitir a los usuarios crear experiencias de administración desde su propio CMS en lugar de ir a la Galería en Studio.

Este inicio rápido proporciona una implementación de muestra que permite a los usuarios:

  1. Obtenga todas las plantillas de experiencia en la página para una cuenta
  2. Crea una experiencia básica a partir de la plantilla seleccionada
  3. Seleccione una lista de reproducción o una lista de videos para agregar a la experiencia
  4. Actualiza la experiencia con los videos / lista de reproducción seleccionados
  5. Publica la experiencia

A continuación se muestra una vista de alto nivel de la lógica de la aplicación.

  • Encuentre todo el código asociado con esta muestra en este Repositorio de GitHub.

Código de proxy

Para crear su propia versión de la aplicación de muestra en esta página, debe crear y alojar su propio proxy. (Los proxies utilizados por Brightcove Learning Services solo aceptan solicitudes de dominios de Brightcove). Puede descargar dos versiones de nuestro código proxy:

  • Esta es una versión general que espera que se pasen las credenciales del cliente con la solicitud.
  • Esta versión le permite guardar sus credenciales de cliente en el propio proxy en las líneas 25-26 (recomendado)
