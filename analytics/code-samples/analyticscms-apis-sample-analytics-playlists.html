Limitaciones

Esta muestra y el código asociado se proporcionan como una guía para su desarrollo de producción. El código no está certificado para uso en producción ni cuenta con respaldo oficial.
  • Debe recuperar los datos de la lista de reproducción y seleccionar una lista de reproducción antes de enviar la solicitud a la API de Analytics.
  • Para las listas de reproducción inteligentes, los datos analíticos se devolverán para los videos que se encuentran actualmente en la lista de reproducción; los datos no se devolverán para los videos que ya no están en la lista de reproducción porque se han desactivado, eliminado o eliminado debido a que la definición de la lista de reproducción ha cambiado.
  • Si alguno de los videos también se publica en reproductores de video únicos o en varias listas de reproducción, deberá especificar el reproductor para obtener datos precisos para una lista de reproducción específica.
  • Los datos devueltos incluyen los datos de videos individuales más valores acumulativos o promedio para la lista de reproducción.
  • Si los datos devueltos no incluyen elementos para todos los videos en la lista de reproducción, no se devolvieron datos para esos videos durante el período de tiempo especificado.

Lógica de la aplicación

Esta es una aplicación híbrida que usa el API de CMS para recuperar la información de la lista de reproducción, y luego API de análisis para recuperar análisis de los videos en la lista de reproducción. Hay un pequeño giro: para la lista de reproducción EXPLÍCITA (manual), los metadatos incluyen los identificadores de video, por lo que podemos llevarlos directamente a la API de Analytics; para las listas de reproducción inteligentes, sin embargo, debemos volver a API de CMS utilizando los criterios de búsqueda, clasificación y límite de la lista de reproducción para obtener la información del video, que luego podemos llevar a la API de Analytics:
