Error de mensajes

Código HTTP Código de error Descripción Resolución
400 BAD_REQUEST Este error indica que la solicitud realizada tuvo un error del lado del cliente. El mensaje generalmente tiene información sobre cuál fue el error. La message Los campos de la respuesta contienen información sobre la causa del error, como `valor no válido para sort parámetro.
400 UNSUPPORTED_FIELD_COMBINATION_ERROR Este error indica que el fields El parámetro de un informe incluía uno o más campos que no son válidos para el tipo de informe. La message Los campos de la respuesta contienen información sobre qué campos no válidos se especificaron.
400 TIME_RANGE_UNAVAILABLE Esta respuesta se devuelve cuando el sistema de procesamiento histórico / por lotes no tiene los datos que solicita.
401 UNAUTHORIZED Este error indica que la autorización (token OAuth2 o LA) utilizada para realizar la solicitud no es válida. Verifique el token de autorización que se está utilizando. Es posible que el token utilizado esté más allá de la ventana de tiempo para la que era válido y se necesita un token más nuevo. Generalmente, cada token caduca después de 5 minutos. El token de autorización se puede especificar como un parámetro de URL. access_token o en el Authorization encabezamiento. Referirse a Administrar las credenciales de la API para más detalles.
404 NOT_FOUND Este error generalmente se devuelve cuando el punto final solicitado no existe. Compruebe el punto final que se solicita.
405 METHOD_NOT_ALLOWED Este error ocurre cuando la solicitud de API se realiza con un método HTTP que no sea GET. Realice todas las solicitudes a la API con HTTP GET.
500 SERVER_ERROR Este error ocurre cuando hay un error interno en el servidor. El servidor puede devolver esto cuando no esté disponible o se enfrente a un problema transitorio. Inténtelo de nuevo en algún tiempo con cierta cantidad de retroceso exponencial. Si el problema persiste, cree un ticket de soporte.
503 PROCESSING Esto significa que el servidor ha recibido la solicitud y la está procesando actualmente. los API de análisis puede devolver este mensaje si encuentra una consulta de larga duración. Una vez finalizada la consulta, se almacenará en la caché del servidor durante un máximo de 5 minutos. Por lo tanto, sugerimos consultar la API 4 minutos después de recibir este error.
