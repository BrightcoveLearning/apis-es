Parámetros de URL

Los informes de la API de Analytics admiten los siguientes parámetros de URL.

Parámetros de URL
Parámetro Descripción Necesario Valores Defecto
account Las cuentas sobre las que desea informar sí uno o más ID de cuenta como una lista delimitada por comas ninguno
dimensions Las dimensiones sobre las que informar sí una o más dimensiones como una lista delimitada por comas (algunas combinaciones no son válidas) ninguno
where Se utiliza para especificar filtros para informes. no {dimension}=={value} - uno o más como una lista delimitada por punto y coma ninguno
limit Número de artículos a devolver no entero positivo 10
offset Número de elementos para omitir no entero positivo 0
sort Campo en el que ordenar los elementos no un campo válido para la solicitud video_view
fields Campos para devolver no varía según la dimensión sobre la que está informando video_view
format Formato para devolver resultados en no json | csv | xlxs json
reconciled Si se incluye, limitará los resultados a datos históricos o en tiempo real no verdadero | falso cierto
from El comienzo del rango de fechas para la solicitud no Una fecha ISO 8601 (AAAA-MM-DD), tiempo de época en milisegundos, la cadena de tiempo completo o fecha relativa (-1 m); solo se permiten fechas dentro de los últimos 32 días para los puntos finales de participación o si reconciliado = falso. 30 días antes de ahora
to El final del rango de fechas para la solicitud no Una fecha ISO 8601 (AAAA-MM-DD), tiempo de época en milisegundos, la cadena ahora o datos relativos (+ 7d); solo se permiten fechas dentro de los últimos 32 días para los puntos finales de participación o si reconciliado = falso. ahora

Campos disponibles

Los siguientes campos se pueden devolver para la dimensión.

  • ad_mode_begin- la cantidad de veces que el jugador ingresó al modo de anuncios
  • ad_mode_complete- la cantidad de veces que el jugador completó el modo de anuncio
  • city- la ciudad donde ocurrieron las impresiones de videos
  • dma- la zona DMA (aplicable solo a EE. UU.)
  • engagement_score- la puntuación media de interacción de un vídeo reproducido en un reproductor
  • play_request- número total de veces que se solicitó la reproducción de video
  • play_rate- la tasa de reproducción promedio (vistas de video divididas por impresiones) de un video
  • player- la identificación del jugador
  • player_load- la cantidad de veces que se cargó un jugador
  • video_impression- la cantidad total de videos cargados en el reproductor
  • video_percent_viewed- la suma del porcentaje de videos vistos en el reproductor
  • video_seconds_viewed- segundos totales de videos vistos en el reproductor
  • video_view- total de veces que comenzó la reproducción de video en el reproductor

Filtrar valores

Si se utiliza como filtro, una dimensión le permite limitar los resultados.

Valores de filtro: Una lista delimitada por comas de nombres de ciudades, por ejemplo, Seattle, Boston, San% 20Francisco

Combinando dimensiones

Para ver con qué dimensiones se puede combinar esto y qué campos están disponibles para la combinación, consulte Dimensión y campos.

Ejemplos de solicitud

Solicitud de dimensión

    https://analytics.api.brightcove.com/v1/data?accounts=1752604059001&dimensions=city

Solicitud de filtro

    https://analytics.api.brightcove.com/v1/data?accounts=1752604059001&dimensions=country&where=city==London,Boston,San%20Francisco&fields=city
