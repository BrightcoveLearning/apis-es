Intervalos de fechas

¿Cómo obtengo datos para un día específico?
Establecer tanto el from y to valores a esa fecha. Por ejemplo: from=2013-05-12&to=2013-05-12. Además, puede consultar por el date dimensión , que se puede combinar con otras dimensiones.
¿Deberían el módulo de análisis y el API de análisis darme datos de identificación para el mismo rango de fechas?
Sí, pero pueden ser diferentes si está comparando datos que son relativamente recientes y obtiene datos de Studio y la API en diferentes momentos:
  1. Los datos tienen menos de 3 días. Los datos analíticos con menos de 3 días de antigüedad son provisionales y están sujetos a cambios en cualquier momento.
  2. Los datos para el mismo período se analizan cuando tenían menos de 32 días y nuevamente después de los 32 días. Después de 32 días, los datos se mueven a un repositorio "histórico" separado, y en ese momento descartamos algunos datos, especialmente los datos de participación detallados. Esto significa que los campos calculados, como la puntuación de participación, pueden cambiar ligeramente, porque el cálculo se realiza en datos menos granulares.

Granularidad de datos

¿Cuál es la granularidad de los datos de Analytics recientes?
Para los 32 días más recientes (incluido el día actual), el API de análisis informa una granularidad de una hora, porque guarda los valores en depósitos por hora. Sin embargo, la hora actual todavía se está llenando, por lo que esto puede dar a la API la apariencia de tener más granularidad que una hora cuando solicita datos que caen en la hora actual.

Por ejemplo:

  1. Si solicita datos de hace 3 horas para las 9:15 y las 9:20, puede obtener el mismo valor si caen en el mismo intervalo de horas; sin embargo, pueden ser diferentes porque todos los datos analíticos son provisionales y están sujetos a cambios hasta que se concilien.
  2. Si solicita datos de hace 10 minutos, y luego espera 5 minutos y vuelve a preguntar, es posible que obtenga un valor diferente incluso si está en el mismo depósito, porque ese depósito aún se está actualizando.
¿Cuál es el intervalo de actualización para los datos analíticos?
El punto final / data actualmente se almacena en caché durante 5 minutos entre consultas, por lo que para las listas de videos con tráfico en la última hora, los intervalos de 5 minutos es el retraso más pequeño que le dará un delta para trabajar.
¿Cuál es la granularidad de los datos históricos de Analytics?
Para fechas anteriores a los últimos 32 días, el API de análisis informa valores de día completo. Esto significa que si lo solicitaste from=1368334306919&to=1378446336919 (desde el domingo 12 de mayo de 2013 a las 04:51:46 GMT hasta el viernes 06 de septiembre de 2013 a las 05:45:36 GMT), obtendría los mismos resultados que si hubiera solicitado from=2013-05-12&to=2013-09-06

Elementos y campos

¿Cuál es la mejor manera de saber cuántos inicios de transmisión hubo en un período para uno o más videos?
Solo mira el video_view métrica: siempre será igual a los inicios de la transmisión. Puede ver esta métrica en todos los informes, y el resumen le mostrará los inicios de transmisión para todos los videos en el rango de fechas consultado. Para ver el inicio de la transmisión para todos los videos en su cuenta, simplemente ejecute esta solicitud: 
    https://analytics.api.brightcove.com/v1/data?accounts={account_ids}&dimensions=video&limit=all&offset=0&fields=all
¿Cómo devuelvo todos los artículos en un Informe?
Selecciona el limit parámetro para el Informe igual a all.
Que datos fields ¿Se devuelven los artículos por defecto?
Por defecto, solo video_view y el campo correspondiente a la dimensión solicitada (p. ej. destination_domain ) se devuelven. Para obtener campos adicionales, configure fields=field_name1,field_name2 para devolver los campos seleccionados.
¿Qué significa si un valor de campo es null?
A null El valor de un campo de datos indica que los datos solicitados no se han procesado. Las razones más probables son las siguientes:
  • Los datos que solicitaste son muy recientes y aún no se han procesado.
  • Los datos que solicitó son muy antiguos y aún no se han importado al sistema de análisis actual.
Cambié el nombre de uno de mis videos. ¿Por qué veo el nombre anterior en los datos de Analytics?
El nuevo nombre se registrará para cualquier nuevo evento de análisis, pero no cambiamos el nombre en los datos históricos; el nombre del video devuelto será el nombre que tenía en el momento en que se vio.
¿Por qué no siempre veo números enteros para los valores de participación?
Es posible que los números de compromiso regresen con puntos decimales. El motivo es que la participación está normalizada, lo que significa que es una proporción de "video_percent_viewed * (video_engagement_25 / video_engagement_sum)", por lo que, según el intervalo de tiempo seleccionado para la consulta, es posible que vea números de punto flotante en los casos en que no se dividen exactamente.
¿Cómo filtro los resultados por reproductor?

Tenga en cuenta lo siguiente cuando utilice where filtros en los reproductores:

  • reproductores de Brightcove - Ingrese la URL del reproductor en este formato: players.brightcove.com/account id/playerID_default
  • reproductores inteligentes: ingrese el ID del reproductor

Reportando

¿Por qué veo 0 GB cuando miro el ancho de banda utilizado por un reproductor?
En el pasado, Brightcove informó el ancho de banda por reproductor, por lo que los clientes que querían desglosar las cosas por reproductor podían usar estas métricas como un medio para asignar costos de ancho de banda. Sin embargo, a medida que avanzamos hacia una entrega más basada en manifiestos (HLS hoy y DASH en el futuro), la naturaleza del video segmentado hará que el seguimiento del ancho de banda por reproductor sea impreciso. Por lo tanto, desaprobaremos el ancho de banda por dimensión de reproductor en el informe de Utilización. Por lo tanto, en el futuro, recomendamos usar los segundos vistos por reproductor en el informe de rendimiento para asignar costos por reproductor.

Diverso

¿Puedo excluir las vistas de los empleados de los informes analíticos?
No hay una manera fácil de hacer esto, y para los videos de alto tráfico, las vistas de los empleados son probablemente un porcentaje tan pequeño que su efecto en la analítica es insignificante. Sin embargo, para videos de poco tráfico en los que cree que es importante hacer esto, la solución más simple es duplicar sus reproductores de producción y hacer que los empleados prueben / vean videos en las copias. Luego, puede usar filtros para crear informes sobre sus reproductores de producción únicamente, utilizando la función Informes personalizados en el módulo de Analytics o la API de Analytics.
