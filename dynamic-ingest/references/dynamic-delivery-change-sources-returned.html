Aviso: cambio de fuentes

Durante marzo de 2019, Brightcove está actualizando la infraestructura de Dynamic Delivery para realizar un pequeño cambio en las fuentes que devuelven las API de reproducción / CMS y, por lo tanto, se entregan a Brightcove Player y a los reproductores SDK nativos.

Para la gran mayoría de los usuarios de Video Cloud, este cambio es completamente transparente. No hay nada que deba hacer. Para algunos clientes que han construido integraciones basándose en la estructura exacta de las fuentes existentes, estas integraciones deberán cambiar.

¿Qué está cambiando?

Actualmente, Dynamic Delivery devuelve 2 sabores (v4 y v5) de HLS cuando entran las solicitudes de la API de reproducción. En algunos casos especiales, generalmente la entrega a un dispositivo específico, las cuentas también están habilitadas para la entrega v3.

Para cada uno de ellos, también devolvemos los siguientes tipos de DRM:

  • V3: Borrar / AES-128 (si su cuenta está habilitada para v3)
  • V4: Claro / AES-128
  • V5: Claro / AES-128 / Fairplay

La única razón por la que entregamos HLSv5 es que es compatible con Fairplay: para Clear y AES-128, el video y los manifiestos que devolvemos son idénticos.

Sin embargo, debido a que las URL para ellos son diferentes (contienen "v4" o "v5"), duplicamos la cantidad de contenido que necesitamos entregar y almacenar en caché, lo cual es un desperdicio e impacta el rendimiento de Dynamic Delivery y SSAI.

Por lo tanto, estamos cambiando la infraestructura para que solo se devuelva el manifiesto de Fairplay para la versión 5, de modo que el conjunto de fuentes ahora se verá así:

  • V3: Borrar / AES-128 (si su cuenta está habilitada para v3)
  • V4: Claro / AES-128
  • V5: Juego limpio

¿Esto afectaría a mis reproductores?

  1. Si usted no hacer utilizar encriptación Clear DRM o AES-128, este cambio No hace diferencia
  2. Si usted hacer use el cifrado Clear DRM o AES-128, pero deje que el reproductor seleccione la fuente automáticamente, este cambio No hace diferencia (el reproductor seleccionará la fuente v4)
  3. Si usted hacer use el cifrado Clear DRM o AES-128, y haya escrito un código personalizado para obligar al reproductor a usar las URL de origen v5, este cambio Como resultado, el reproductor no tiene una fuente reproducible. - la solución es permitir que el reproductor seleccione automáticamente la fuente o modifique su código para seleccionar la fuente v4 en su lugar.
