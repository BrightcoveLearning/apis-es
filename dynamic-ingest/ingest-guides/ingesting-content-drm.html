Descripción general

DRM (Gestión de derechos digitales) protege su contenido de video al cifrar los datos de video y desbloquearlos según las políticas de licencia. Cuando el contenido de video se carga en un reproductor de Brightcove, el reproductor vuelve a llamar a un servidor de licencias y obtiene permiso para reproducir el video. Esta transacción es transparente para los espectadores.

Para obtener una descripción general de la compatibilidad con DRM y Video Cloud, consulte Descripción general: Gestión de derechos digitales en Video Cloud.

Notas:
  • Para utilizar el cifrado DRM o HLS, su cuenta debe estar habilitada para estas funciones. Comuníquese con su gerente de cuenta para obtener más información.
  • Cuando los videos están habilitados para DRM, las fuentes / representaciones de MP4 se ocultan automáticamente (lo que significa que existen, pero no se envían al reproductor como fuentes) en Studio, la CMS API y la Playback API.

  • Actualmente, DRM es compatible con los siguientes navegadores de sistemas operativos móviles:

    • iOS Safari (11.2+)
    • Android Chrome

    Widevine es compatible con aplicaciones de Android y FairPlay es compatible con aplicaciones de iOS. Para obtener detalles sobre la creación de aplicaciones móviles, consulte la SDK de reproductores nativos de Brightcove Página de inicio.

  • En la actualidad, Brightcove no admite la reproducción DRM de solo vídeo o solo audio.

DRM para entrega dinámica

Si su cuenta está habilitada para Entrega dinámica y ha sido habilitado para DRM por su administrador de cuenta, todos los videos recién cargados estarán protegidos. Si desea aplicar protección a los Entrega dinámica vídeos, póngase en contacto con el soporte de Brightcove.

Si tiene videos más antiguos en una cuenta de Dynamic Delivery que fueron ingeridos usando el sistema de ingesta heredado (no Dynamic Delivery), deberán volver a transcodificarse mediante un perfil de Dynamic Delivery antes de que se aplique DRM. Vídeos existentes que ya se han ingerido para Dynamic Delivery no es necesario volver a codificar , pero como se indicó anteriormente, debe comunicarse con el Soporte y pedirles que apliquen protección DRM a esos videos.

Si su cuenta está habilitada para DRM y desea entregar algunos títulos sin DRM, configure el drm_disabled bandera a través de CMS API para true usando un Actualizar solicitud de video.

Solicitud de muestra

Pedido

La solicitud es PATCH solicitud de:

              https://cms.api.brightcove.com/v1/accounts/account_id/videos/video_id

Cuerpo de la solicitud

              {
                  "drm_disabled": true
              }
