    Descripción general de la entrega dinámica

    En este tema se proporciona una descripción general de la entrega dinámica. Tenga en cuenta que esta guía es para la mayoría de las cuentas de Video Cloud que se encuentran en el sistema de ingesta de entrega dinámica. Aquellos que tengan cuentas más antiguas aún en el sistema de ingesta heredado deben ir a la versión Dynamic Ingest API heredada en su lugar.

    La entrega de medios a diferentes dispositivos (en televisores de escritorio, móviles y conectados) requiere una combinación de tecnologías en constante cambio:

    • Códecs de audio y vídeo
    • Varios tipos de empaquetado para streaming de vídeo
    • Varios esquemas de protección de contenido para proteger el contenido de ser copiado

    Sin el uso de Dynamic Delivery, la transmisión multiplataforma a menudo requiere almacenar el mismo contenido varias veces para admitir la entrega a todas las plataformas de destino. Dynamic Delivery permite a los clientes de Brightcove entregar medios a una amplia gama de dispositivos sin el coste y la sobrecarga operativa de volver a empaquetar y almacenar contenido varias veces para cada dispositivo compatible y caso de uso. Dynamic Delivery crea el paquete de contenido óptimo justo a tiempo para su entrega y transmisión a todos los dispositivos compatibles y casos de uso. Esto también ayuda a proteger su arquitectura de entrega a medida que la tecnología y el público evolucionan.

    Con Dynamic Delivery, los clientes proporcionan un único archivo de vídeo (entreplanta) de alta calidad. Durante el proceso de ingesta, creamos varios niveles de calidad de este archivo adaptados a una gama de diferentes tamaños de pantalla y condiciones de red. Cuando un espectador pulsa reproducir, Video Cloud utiliza información sobre el dispositivo solicitante para determinar el tipo de tecnología de transmisión esperada, por ejemplo, el tipo de empaquetado de secuencias (HLS vs. DASH) (o MP4 si el dispositivo no admite ningún tipo de transmisión HTTP), el DRM requerido (por ejemplo, Widevine, Fairplay, Playready) y contenido auxiliar, como subtítulos opcionales o pistas de audio alternativas. A continuación, Dynamic Delivery utiliza esta información para entregar el paquete de contenido apropiado (es decir, el tipo de flujo) para el dispositivo solicitante en tiempo real.

    Descripción de la arquitectura

    1. Cuando se carga un archivo de origen, el proceso de transcodificación utilizará el perfil de ingesta seleccionado para determinar el conjunto de copias que se van a crear.
    2. Las copias se almacenan como un conjunto de archivos MP4 contiguos y fragmentados de varias velocidades de bits y resoluciones que son agnósticos de cualquier formato de entrega específico.
    3. Cuando se solicita un vídeo, el proceso de entrega dinámica realiza el empaquetado Just-in-Time o JIT, lo que significa que empaquetará dinámicamente el contenido en el formato adecuado según los requisitos especificados. Este empaquetado tendrá en cuenta el formato de entrega específico necesario, las pistas de audio y subtítulos necesarios, así como cualquier cifrado o DRM necesario.
    4. Cuando el dispositivo de reproducción utiliza el manifiesto devuelto por la API de reproducción para solicitar la reproducción multimedia, realizamos empaquetado justo a tiempo en los archivos MP4 fragmentados para proporcionar el formato de segmento adecuado, la duración y el cifrado. Estos fragmentos se almacenan en caché en una CDN, de modo que la entrega de vídeo sigue siendo de alto rendimiento.

    El proceso de entrega dinámica se diseñó para ser un servicio completamente gestionado y basado en la nube. Todos los componentes se ejecutan en infraestructura de nube en varias regiones para asegurar alta disponibilidad, redundancia y escalabilidad.

    ¿Qué formatos de vídeo produce Dynamic Delivery?

    Entrega dinámica produce los siguientes orígenes según sea necesario:

    • HLS (versiones 3, 4 y 5)
    • Smooth
    • PIZCA
    • MP4 (si se especifican copias progresivas en el perfil de ingesta)

    Se crean los siguientes DRM-PackageRenditions (dependiendo de la protección de contenido para la que esté habilitada su cuenta):

    • HLS + Fairplay
    • DASH + Widevine
    • DASH + Playready
    • Suave + Playready

    Proceso de migración

    A partir del martes 17 de octubre de 2017, todas las cuentas nuevas de Video Cloud que se crean están configuradas para utilizar la Entrega dinámica de forma predeterminada. Esto incluye cuentas de prueba, cuentas de nuevos editores y nuevas subcuentas para editores existentes. Con este cambio, todos los nuevos editores y clientes de prueba se beneficiarán automáticamente de todas las funciones de Dynamic Delivery.

    Para los clientes existentes, la migración es un proceso de dos pasos. En primer lugar, las nuevas cargas se transicionan para utilizar la entrega dinámica (migración diaria) y, en segundo lugar, el contenido existente se migra de la plataforma de entrega heredada a Dynamic Delivery (migración de catálogo posterior).

    Migración de un día adelante

    El primer paso para pasar a Entrega dinámica es la transición de las nuevas cargas para utilizar la Entrega dinámica. Habilitar su cuenta para Dynamic Delivery es una tarea sencilla que puede ser completada por el Servicio de Atención al Cliente. No hay peligro en habilitar una cuenta de Video Cloud existente para Dynamic Delivery, ya que no modifica ningún contenido existente. Sólo el nuevo sistema utilizará el contenido que se ingiera explícitamente en Dynamic Delivery.

    Desde el punto de vista del flujo de trabajo, un par de cosas aparecerán de manera diferente dentro de Video Cloud Studio. Primero, al cargar vídeos con el módulo Subir, ahora verás perfiles de ingesta de entrega dinámica en tu cuenta.

    El módulo Media también mostrará un icono de nube verde () junto a los títulos de vídeo ingeridos mediante un perfil de entrega dinámica. Los vídeos que se ingieren utilizando un perfil de ingesta heredado mostrarán un icono gris ().

    Además, el panel Fuentes de vídeo del módulo multimedia aparecerá ligeramente diferente debido a que las copias se almacenan en un estado agnóstico de formato.

    Funcionalmente, todo lo demás sigue siendo el mismo. Puede cargar a través de todos los mecanismos estándar (Video Cloud Studio, API de ingesta dinámica, conectores CMS, etc.) y la reproducción permanece sin cambios en todos sus dispositivos de reproducción estándar.

    Una vez que se ha ingerido un vídeo mediante Entrega dinámica, si se vuelve a transcodificar el vídeo, se debe seleccionar un perfil de entrega dinámica. Si desea utilizar un perfil de ingesta heredado, el vídeo debe volver a cargarse.

    Nota: Dynamic Delivery introduce el concepto de ingesta y entrega regionalizadas. En los sistemas de entrega anteriores, todo el procesamiento de la ingestión y el origen de la entrega residían en los Estados Unidos. Con Dynamic Delivery, Brightcove puede configurar una cuenta en la que se produce la ingestión y donde el origen reside publicador por editor para mejorar el rendimiento. Todas las cuentas de Video Cloud se establecerán de forma predeterminada en EE. UU. Este, pero se pueden volver a configurar en su región local. Tenga en cuenta que hay implicaciones de costo cuando se muda a otra región. Póngase en contacto con su administrador de cuentas de Brightcove para obtener más información.

    Migración de catálogo posterior

    Una vez que tenga nuevas ingestas utilizando Dynamic Delivery, podremos migrar su catálogo posterior. Esto se puede hacer de una de dos maneras:

    Herramienta de migración Transmux

    La herramienta de migración transmux mueve el contenido de Video Cloud de la entrega heredada a la entrega dinámica sin necesidad de retranscodificación. Esto le permite migrar el contenido mucho más rápido y más fácil que volver a transcodificar contenido. Como tal, debería ser el método preferido de migración para la mayoría de los clientes. La herramienta transmux transformará las copias existentes en tu cuenta para que sean compatibles con Entrega dinámica. Para migrar tu contenido a través de la herramienta transmux, ponte en contacto con tu gestor de cuentas.

    Re-transcodificación

    Existen algunos criterios que hacen que un título no sea elegible para migrar a través de la herramienta transmux y, como tal, requerirán que los títulos se vuelvan a codificar para migrarlos. Los títulos deberán volver a codificarse si:

    • Desea utilizar la codificación consciente de contexto
    • Desea realizar la inserción de anuncios en el lado del servidor (SSAI)
    • El título no tiene copias HLS
    • Tu título tiene DRM actualmente (HlSE es aceptable)

    Si necesita volver a transcodificar títulos, deberá administrar ese proceso por su cuenta (siguiendo nuestras prácticas recomendadas) o interactuar con su equipo de cuentas para servicios de migración de pago. Nota: Es posible transmux algunos títulos dentro de una cuenta y volver a transcodificar otros.

    Preguntas más frecuentes

    ¿Qué es la entrega dinámica?

    Dynamic Delivery es un conjunto de tecnologías que conforman la plataforma de entrega de medios de última generación de Brightcove. Dynamic Delivery está diseñado para satisfacer los requisitos actuales y futuros de los editores y se convertirá en la base de la forma en que todos los productos Brightcove entregan medios.

    ¿Cuáles son los beneficios de la entrega dinámica?

    • Mayor flexibilidad y agilidad para la entrega: el contenido se empaqueta dinámicamente para su entrega a una serie de puntos finales (navegadores web, teléfonos inteligentes, tabletas, televisores conectados y cajas de transmisión). Dynamic Delivery está diseñado para ser capaz de utilizar múltiples CDN para entregar contenido, lo que aumenta la elección del cliente.
    • Prueba futura: puesto que el contenido se empaqueta dinámicamente, Brightcove puede agregar nuevos formatos sobre la marcha. A medida que sale una nueva versión de HLS, DASH, etc., o incluso un nuevo formato, simplemente agregamos soporte a nivel de plataforma y los editores pueden empezar a usarlo sin volver a transcodificar el contenido existente.
    • Reducción del espacio de almacenamiento: los activos de contenido se almacenan una vez y se crean copias descendentes para su entrega cuando sea necesario, en lugar de almacenar el contenido varias veces.
    • Rendimiento mejorado a través de la distribución geográfica: Dynamic Delivery está diseñado para una arquitectura distribuida a nivel regional y de alta disponibilidad. Los editores se benefician de un mejor rendimiento de reproducción y una mayor resistencia.
    • Compatibilidad integrada para la inserción de anuncios en el lado del servidor (SSAI): la entrega dinámica permitirá insertar anuncios u otro contenido desde el servidor. Esto proporciona una experiencia de reproducción perfecta para los espectadores y ofrece a los editores una mayor resistencia frente a los bloqueadores de anuncios, todo desde Video Cloud.

    Después de ingerir o transcodificar un vídeo usando un perfil de entrega dinámica, ¿puedo retranscodificarlo usando un perfil de ingesta heredado?

    No, los vídeos de entrega dinámica no se pueden retranscodificar utilizando perfiles de ingesta heredados.

    ¿ Hay alguna consideración de precios con Dynamic Delivery?

    No, el acceso a Dynamic Delivery está incluido en todas las licencias de plataforma base. Algunas características de la entrega dinámica, como la inserción de anuncios en el lado del servidor (SSAI) y la codificación consciente del contexto (CAE), son complementos con costos asociados. Consulte con su gestor de cuentas para obtener más información sobre los precios.

    ¿ Dynamic Delivery admite editores de CDN BYO?

    Dynamic Delivery permite a los editores aprovechar sus propias relaciones de CDN; sin embargo, todos los activos de vídeo deben ser administrados por Brightcove Origin. Póngase en contacto con su administrador de cuenta para obtener más información.

    ¿Qué regiones admite Dynamic Delivery para la ingesta y el origen?

    Dynamic Delivery admite la ingestión y el origen en las siguientes regiones de AWS:

    • us-east-1 (Virginia) - predeterminado para todas las cuentas
    • eu-west-1 (Dublín)
    • ap-northeast-1 (Tokio)
    • ap-southeast-1 (Singapur)
    • ap-southeast-2 (Sídney)

    Tenga en cuenta que hay implicaciones de coste al trasladar la ingestión y el origen a otra región. Póngase en contacto con su administrador de cuentas de Brightcove para obtener más información.

    ¿Brightcove trasladará todos los editores a Dynamic Delivery?

    Sí, todos los editores nuevos (tanto Video Cloud como Once) se configurarán con Dynamic Delivery. Durante el próximo año, Brightcove trabajará para trasladar a todos los editores a Dynamic Delivery y eventualmente se convertirán en tecnologías de entrega heredadas. No hay fecha asociada a esta migración, pero las compartiremos a medida que estén disponibles. Además, todas las funciones nuevas solo se agregarán a la entrega dinámica.

    ¿Cómo puedo saber si mi cuenta ya está habilitada para Dynamic Delivery?

    La forma más fácil de saber si tu cuenta está habilitada para Entrega dinámica es navegar hasta el módulo de carga y abrir el menú desplegable «Perfil de ingesta». Si ves el subtítulo «Entrega dinámica» significa que tu cuenta está habilitada para Entrega dinámica.
