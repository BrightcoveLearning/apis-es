Introducción

El servicio de autorización de reproducción (PAS) de Brightcove ofrece un nivel adicional de seguridad cuando se utiliza Dynamic Delivery con contenido protegido por DRM o HTTP Live Streaming Encryption (HLSe).

Esta función le da control sobre la reproducción de su contenido al requerir un token para autorizar la reproducción. Se requiere un token JWT válido para obtener una clave, necesaria para descifrar HLS cifrado, o una licencia, necesaria para descifrar contenido protegido por DRM.

Si no está familiarizado con esta función, consulte la Descripción general: DRM con servicio de autorización de reproducción documento.

Flujo de trabajo

Para utilizar el servicio de autorización de reproducción, siga estos pasos:

  1. Crear un Token web JSON
  2. Configurar el reproductor para usar el token

Crear un Token web JSON

Para crear un token para usar al comunicarse con la API de reproducción de Brightcove, consulte la Creando un Token web JSON (JWT) documento.

Configurar el reproductor

El token se pasa con la solicitud de reproducción. Si el token no es válido o ha caducado, se restringirá el acceso al contenido. El reproductor debe estar configurado para usar el token.

Reproductor web

Para obtener detalles sobre la configuración del reproductor web Brightcove, consulte la Uso del servicio de autorización de reproducción con Brightcove Player documento.

Reproductor nativo de Android

Para obtener detalles sobre la configuración del reproductor nativo para Android, consulte la Uso del servicio de autorización de reproducción con los SDK nativos documento.

Reproductor nativo de iOS

Para obtener detalles sobre la configuración del reproductor nativo para iOS, consulte la Uso del servicio de autorización de reproducción con los SDK nativos documento.

Limitaciones

Las siguientes limitaciones se aplican a esta función:

  • Para HLSe (AES-128), el token se agregará a la solicitud del manifiesto maestro y se propagará a la URL de la clave de cifrado. Debido a que estas URL se adjuntan al manifiesto maestro, los clientes deben considerar restringir la cantidad de usos para minimizar la exposición de su contenido.
  • Para HLSe (AES-128), los reproductores realizarán múltiples solicitudes al reproducir un video, generalmente una por interpretación. La reclamo maxu debe establecerse lo suficientemente alto para tener en cuenta estas solicitudes. Debe considerar afirmaciones adicionales para minimizar la exposición de su contenido. El cambio de versiones requerirá una nueva licencia.
  • Seguimiento de la Reclamaciones maxu y maxip eventualmente es consistente. Esto significa que si un token que usa estos reclamos se reutiliza muchas veces en rápida sucesión, es posible ver que el token se acepta más veces de lo que declaran esos reclamos. El seguimiento se vuelve consistente en unos pocos segundos y después de esa ventana, el token se bloqueará como se esperaba.
