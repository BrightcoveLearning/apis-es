    Aplicación de las normas de entrega

    En este tema, aprenderá cómo implementar las reglas de entrega de Brightcove, que le permiten personalizar cómo se entregan los medios para cumplir sus objetivos empresariales.
    Esta función está disponible para un conjunto específico de clientes con acceso a la fase de disponibilidad limitada de Reglas de entrega.

    Resumen

    Los editores a menudo necesitan modificar las características del contenido de sus manifiestos multimedia por razones técnicas y comerciales. Para satisfacer estas necesidades, Brightcove ha creado un marco en el que se pueden aplicar reglas que modificarán el contenido del manifiesto devuelto por Dynamic Delivery.

    Las Reglas de Entrega se componen de condiciones y acciones.

    • Condiciones - Condiciones «Si» que desencadenan una determinada acción
    • Acciones - Parámetros «Entonces» que definen cómo se modifica el manifiesto

    Para obtener más información acerca de las Reglas de Entrega, consulte lo siguiente:

    • Visión general: Reglas de entrega
    • Uso de reglas de entrega con Brightcove Player
    • Uso de reglas de entrega con los SDK nativos
    • Descripción general de la entrega dinámica

    Configuración de la cuenta

    Póngase en contacto con su administrador de cuenta para obtener información sobre los precios y activar esta función para su cuenta.

    Aplicación de reglas

    El siguiente diagrama de flujo muestra cómo se aplican las reglas de entrega.

    Condiciones

    Las condiciones «si», soportadas en la primera fase, incluyen las siguientes:

    • Solicitud explícita : un ID de configuración es el identificador de una acción que ha creado. Utilizará este id para pasar a la API de reproducción en tiempo de ejecución. Este id se asigna a valores de configuración que permiten a los servicios subyacentes tomar decisiones aprovechadas por la API de reproducción para devolver los datos de respuesta adecuados.

    • Grupo de dispositivos : tipo de dispositivo detectado por el análisis del agente de usuario. Los grupos admitidos actualmente incluyen:
      • android
      • appletv
      • iPad
      • iphone
      • sobremesa de cromo
      • desktop-firefox
      • safari de sobremesa
      • fetchtv
      • ios-otro
      • roku-7/8
      • smartphone-genérico
      • tableta-android
      • tablet-Kindle
      • desconocido
       
    • Geografía - Ubicación física del dispositivo solicitante:
      • Continente
      • País

    Acciones

    Las acciones, o parámetros «entonces», admitidos en la primera fase incluyen las siguientes:

    • Media Delivery CDN: desde las CDN configuradas para su uso dentro de una cuenta de entrega dinámica determinada, la CDN que se utilizará para la entrega de segmentos de medios.

    • Características de copia - Filtros basados en las características de las copias de medios subyacentes. Son las siguientes:
      • Velocidad de bits mínima de vídeo
      • Velocidad de bits máxima de vídeo
      • Resolución mínima de vídeo
      • Máxima resolución de vídeo
      • Número máximo de copias de audio
      • Número máximo de copias de vídeo
      • Primera velocidad de bits de vídeo
      • Discontinuidades

    Ámbito

    Las acciones se pueden aplicar en dos niveles:

    • Cuenta - Acciones que se aplican a todas las solicitudes realizadas a la cuenta de un cliente específico
    • Solicitud específica - Acciones que se invocan en una solicitud específica

    Las acciones se aplicarán en el orden indicado anteriormente. Las acciones posteriores anularán las anteriores.

    Definición de reglas

    Utilice la API de reglas de entrega para personalizar la entrega de medios.

    API de reglas de entrega

    La API de reglas de entrega le permite definir condiciones y acciones para controlar la entrega de medios.

    URL base

    La URL base para la API es: 

    https://delivery-rules.api.brightcove.com

    Ruta de la cuenta

    En todos los casos, se solicitarán una cuenta de Video Cloud específica. Por lo tanto, siempre agregará las cuentas de término seguidas de su ID de cuenta a la URL base: 

    https://delivery-rules.api.brightcove.com/accounts/{accountID}

    Autorización

    Se requiere un token de acceso para solicitudes y debe estar presente en el encabezado Autorización። 

    Authorization: Bearer {access_token}

    El token de acceso es un token de acceso temporal de OAuth2 que debe obtenerse del servicio OAuth de Brightcove. Para obtener información detallada sobre cómo obtener credenciales de cliente y usarlas para recuperar tokens de acceso, consulte la descripción general de OAuth de Brightcove.

    Permisos

    Las solicitudes a la API de derechos de reproducción deben realizarse desde credenciales de cliente con los siguientes permisos:

    • video-cloud/delivery-rules/read
    • video-cloud/delivery-rules/all

    Administrar reglas

    La API de reglas de entrega admite las siguientes solicitudes. Para obtener más información sobre la API, consulte la Referencia de la API de reglas de entrega.

    Obtener reglas de entrega

    Utilice la siguiente solicitud GET para obtener la versión más reciente de Reglas de entrega de una cuenta. 

    GET /accounts/{accountID}

    Limitaciones

    Existen algunas limitaciones conocidas al usar Reglas de entrega:

    • Contenido multimedia: las reglas de entrega no funcionarán con vídeos Smooth o PMP4.
    • Solo audio : las reglas de entrega con anuncios del lado del servidor (SSAI) habilitados requieren contenido de vídeo y audio. Esta es una restricción de SSAI.
