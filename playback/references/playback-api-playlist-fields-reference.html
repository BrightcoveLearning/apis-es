Campos primarios

Tabla 1: Campos primarios
Campo Tipo Defecto Descripción
id cuerda ninguno la identificación de la lista de reproducción
account_id cuerda ninguno el id de la cuenta
created_at Cadena de fecha y hora ISO 8601 ninguno se agregó un video de fecha y hora a la cuenta; ejemplo: "2014-12-09T06: 07: 11.877Z"
description cuerda null la breve descripción del video - 128 caracteres de un solo byte como máximo
favorite booleano false si la lista de reproducción está incluida en favoritos
limit número 100 número de videos que puede contener la lista de reproducción (número entero, 1 - 100) devuelto solo para listas de reproducción inteligentes
name cuerda ninguno Título de la lista de reproducción: campo obligatorio
reference_id cuerda null cualquier valor que sea único dentro de la cuenta
search cuerda null cadena de búsqueda utilizada para definir los videos para una lista de reproducción inteligente (ejemplo: +tags:"bird","birds")
type cuerda el tipo de lista de reproducción Ver nota[1] debajo
updated_at Cadena de fecha y hora ISO 8601 ninguno la lista de reproducción de fecha y hora se modificó por última vez; ejemplo: "2015-01-13T17: 45: 21.977Z"
videos formación gama de objetos de video para los videos en la lista de reproducción solo los primeros 20 videos se devuelven para las listas de reproducción

Notas

  • [1]

    La type El valor solo puede ser uno de: EXPLÍCITO, ACTIVATED_OLDEST_TO_NEWEST , ACTIVATED_NEWEST_TO_OLDEST , ALPHABETICAL , PLAYS_TOTAL , PLAYS_TRAILING_WEEK , START_DATE_OLDEST_TO_NEWEST,START_DATE_NEWEST_TO_OLDEST

    EXPLICIT significa que es una lista de reproducción manual; todos los demás tipos son listas de reproducción inteligentes con nombres que corresponden a su esquema de ordenación.
