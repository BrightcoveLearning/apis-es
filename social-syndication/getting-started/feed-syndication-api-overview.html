La API de fuentes de distribución de Brightcove es una API pública para recuperar fuentes de MRSS creadas con el API de configuración de distribución.

La API de configuración de distribución también se puede utilizar para recuperar feeds simplemente abriendo el syndication_url devuelto en el objeto de distribución en un navegador, pero la API de feeds es diferente en el sentido de que no requiere autenticación: cualquiera que tenga la URL correcta puede recuperar el feed.

Los casos de uso principales para recuperar feeds mediante una API u otra son los siguientes:

  • Si solo desea alojar el feed usted mismo, para los suscriptores, por ejemplo, entonces usaría la API de configuración de distribución para recuperar el feed.
  • Si tiene socios de distribución a los que desea darles el feed para que lo utilicen y alojen de la forma que deseen, puede proporcionar la URL de la API del feed con syndication_id para recuperar feeds directamente.

Referencia de API / URL base

La Referencia de la API de feeds contiene detalles completos sobre las solicitudes y respuestas disponibles.

La URL base es:

https://social.feeds.brightcove.com/v1

Autenticación

El acceso a la API de feeds no no requieren autenticación, ya que los feeds deben ser de acceso público.

Operaciones

Solo hay una operación para la API de feeds: OBTENGA un feed basado en syndication_id.

Solicitar URL

    https://social.feeds.brightcove.com/v1/accounts/{account_id}/mrss/syndications/{syndication_id}/feed

Método: OBTENER

Parámetros de consulta

Todos los parámetros de consulta son opcionales.

limit
Número máximo de elementos para incluir en la respuesta del feed. Rango permitido de 1 a 100. Defecto: 100.
offset
Número de elementos que se deben omitir al generar el feed devuelto. Defecto: 0.
updated_after
Una cadena de fecha y hora ISO-8601 (por ejemplo, 2016-03-16T15: 42: 06.000Z o 2016-03-16T10:47:29-05:00 ) que limitará los resultados del feed a los activos actualizados después de la fecha indicada.

Cuerpo de la solicitud: ninguno

¡Esto es importante! si incluye un cuerpo de solicitud, se devolverá un error.

Tipo de respuesta: XML
