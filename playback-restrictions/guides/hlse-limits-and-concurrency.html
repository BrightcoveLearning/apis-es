Esta función está disponible para un conjunto específico de clientes con acceso a la fase de disponibilidad limitada de las restricciones de reproducción de Brightcove.

Introducción

Si desea limitar su contenido por usuario, pero no necesita o no desea integrar DRM, entonces el uso de HLSe puede ser la solución para usted.

El cifrado estándar con AES-128 (HLSe) le permite cifrar su contenido a través de Internet. No es tan seguro como usar DRM, ya que utiliza una clave de descifrado de acceso público.

Usando HLSe con Protección de clave de licencia de Brightcove , le permite autorizar solicitudes de reproducción basadas en información del usuario o avanzada para permitir o denegar el acceso a las claves de cifrado y mejorar la seguridad del cifrado estándar.

La protección de clave de licencia se centra en la protección de solicitudes de licencia DRM, así como en el descifrado AES-128; Las afirmaciones de JWT permiten que la protección de clave de licencia admita restricciones a nivel de usuario. Esto significa que puede restringir la reproducción a un agente de usuario, un número máximo de direcciones IP diferentes o un número máximo de usuarios para cada token.

Si es nuevo en esta función, consulte la Descripción general: Restricciones de reproducción de Brightcove documento.

Empezando

Configura tu cuenta y prepara tu contenido.

  1. Comuníquese con su administrador de cuentas para habilitar su cuenta para HLSe y restricciones de reproducción.
  2. Una vez que HLSe esté habilitado para su cuenta, ingiera o vuelva a transcodificar los videos que desea proteger.

Flujo de trabajo

Para establecer límites de dispositivos o transmitir simultaneidad con HLSe, siga estos pasos:

  1. Crear un Token web JSON (JWT) por usuario.

    Para cada sesión de usuario, debe crear un JWT. Esto debe generarse en base a las restricciones deseadas con diferentes reclamos.

    • Para conocer los límites de dispositivos, consulte la Establecer límites de dispositivo sección.

    • Para la concurrencia de transmisiones, consulte la Configurar la simultaneidad de la transmisión sección.

  2. Configure su reproductor para utilizar protecciones de clave de licencia.

    Para obtener más detalles, consulte la Uso de la protección de claves de licencia sección de la descripción general: Restricciones de reproducción de Brightcove documento.

Establecer límites de dispositivo

Con la protección de clave de licencia, puede limitar la reproducción a un número determinado de dispositivos por usuario.

Reclamación (es

  • maxip:

    Para el contenido de HLSe, el maxip Claim define cuántas direcciones IP diferentes se pueden transmitir al mismo tiempo.

Ejemplo

Este ejemplo crea un JWT para restringir la reproducción a una dirección IP por usuario (un dispositivo por usuario):

  1. Genere un par de claves pública-privada.

    Para obtener más detalles, consulte la Generar par de claves público-privadas sección del documento Creación de un token web JSON (JWT).

  2. Registre su clave pública.

    Para obtener más detalles, consulte la Usar API de autenticación documento.

  3. Cree un token web JSON.

    Aquí hay un ejemplo usando bash. Asegúrate de agregar el "maxip":1 afirman restringir la reproducción a una dirección IP (dispositivo) por usuario.

    #! /usr/bin/env bash
  # Static header fields.
  HEADER='{
    "type": "JWT",
    "alg": "RS256"
  }'
  
  payload='{
    "pkid": "your policy key",
    "accid": "your account id",
    "maxip": 1
  }'
  
  # Use jq to set the dynamic `iat` and `exp`
  # fields on the payload using the current time.
  # `iat` is set to now, and `exp` is now + 1 second.
  PAYLOAD=$(
    echo "${payload}" | jq --arg time_str "$(date +%s)" \
    '
    ($time_str | tonumber) as $time_num
    | .iat=$time_num
    | .exp=($time_num + 60 * 60)
    '
  )
  
  function b64enc() { openssl enc -base64 -A | tr '+/' '-_' | tr -d '='; }
  
  function rs_sign() { openssl dgst -binary -sha256 -sign {your_private_key_path} ; }
  
  JWT_HDR_B64="$(echo -n "$HEADER" | b64enc)"
  JWT_PAY_B64="$(echo -n "$PAYLOAD" | b64enc)"
  UNSIGNED_JWT="$JWT_HDR_B64.$JWT_PAY_B64"
  SIGNATURE=$(echo -n "$UNSIGNED_JWT" | rs_sign | b64enc)
  
  echo "$UNSIGNED_JWT.$SIGNATURE"
