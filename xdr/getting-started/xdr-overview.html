Descripción general

La función de reanudación entre dispositivos (XDR) de Brightcove permite a los espectadores comenzar a ver videos en un dispositivo, pausar el video y luego continuar viéndolo más tarde en otro dispositivo. Cuando los espectadores reanuden la reproducción, seguirán viendo donde la dejaron. Esto proporciona una experiencia de visualización perfecta para un fragmento de contenido.

El servicio XDR almacena la actividad de visualización, incluida la posición de juego más reciente. Esta información se recupera para reanudar la reproducción en cualquier dispositivo.

Función XDR de Brightcove:

  • Se integra con el sitio web de su cliente y las aplicaciones de dispositivos móviles
  • Utiliza API basadas en REST y llamadas en idiomas nativos en dispositivos móviles y de escritorio
  • Funciona con contenido almacenado en su biblioteca de Video Cloud
  • Conserva la información de la posición de juego durante 90 días
  • Cumple con el RGPD y la COPPA
Tenga en cuenta que hay un retraso entre el envío de datos XDR y la disponibilidad de esos datos a través de la API de hasta 30 segundos.

Requisitos

Se requiere lo siguiente para utilizar el currículum multidispositivo:

  • Debe utilizar la última versión de Brightcove Player y los SDK nativos que admiten la reanudación entre dispositivos.
  • Necesitará una aplicación del lado del servidor para realizar la solicitud de API REST para obtener la posición de reproducción.

Configuracion de cuenta

Comuníquese con su administrador de cuentas para habilitar su cuenta para reanudar entre dispositivos.

Flujo del proceso

Esta función permite a los espectadores comenzar a mirar en un dispositivo, pausarlo y continuar mirando en otro dispositivo.

Paso 1

Mientras un espectador mira un video, el reproductor registra continuamente la posición de la cabeza lectora del video que se está viendo. Esta información se almacena en la API XDR.

Paso 2

Cuando el espectador reanuda la visualización del video en un dispositivo diferente, la posición del cabezal de reproducción se recupera de la API de XDR. La reproducción se reanuda donde la dejó el espectador.

Implementación

Para implementar el currículum en varios dispositivos, siga estos pasos:

  1. Aunque los datos del espectador se envían a Brightcove Analytics automáticamente, debe configurar el identificador del espectador en su aplicación. Para obtener más información, consulte los documentos vinculados en el paso 3.

    La información enviada y almacenada en la API XDR incluirá:

    • account_id- ID de cuenta de Video Cloud
    • viewer_id- ID del espectador cuyas cabezas de reproducción se están guardando
    • video_id- ID de Video Cloud de los videos vistos por el espectador

    • playhead_seconds- Posición actual del cabezal de reproducción en segundos

  2. Obtenga la posición de reproducción de un espectador haciendo una solicitud a la API de XDR. Necesitará una aplicación del lado del servidor para realizar la solicitud de API REST para la posición de reproducción.

    La solicitud a la API XDR devuelve la posición de reproducción en segundos.

    Para obtener más detalles, consulte lo siguiente:

    • Obtener la posición de reproducción de la API XDR
    • Referencia de la API de reanudación entre dispositivos (XDR)

  3. Reanude la reproducción en su aplicación. Para obtener más detalles, consulte lo siguiente:

    • Currículum vitae multidispositivo con Brightcove Player
    • Currículum vitae multidispositivo con los SDK nativos
