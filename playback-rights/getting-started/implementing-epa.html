    Implementación del servicio de administración de derechos de reproducción

    En este tema, aprenderá a administrar la reproducción de vídeo mediante el Servicio de administración de derechos de reproducción de Brightcove.
    Esta función está disponible para un conjunto específico de clientes con acceso a la fase de disponibilidad limitada del Servicio de administración de derechos de reproducción.

    Introducción

    El servicio de gestión de derechos de reproducción de Brightcove proporciona una forma escalable y expresiva de gestionar la reproducción de vídeo.

    Si no está familiarizado con esta función, consulte la descripción general: Gestión del documento Derechos de reproducción.

    Proceso de validación

    Los derechos de reproducción se aplican en orden de especificidad y coincidencia. Una regla de permiso niega el resto de las reglas ya que son menos específicas que la que permite la regla.

    Es posible que desee permitir una IP específica para evitar una regla de país para esa ip. También es posible que desee bloquear una IP diferente que normalmente estaría permitida por la restricción del país. Por lo tanto, puede tener sentido tener ambos block-ips y allow-ips en la misma definición de derechos de reproducción. Lo mismo se aplica a otras reglas.

    Puede tener reglas de permiso y bloqueo para la mayoría de los derechos. Los países son los únicos en los que tal vez no tenga sentido tener ambos.

    Los siguientes diagramas de flujo muestran cómo funciona el proceso de validación.

    Flujo de trabajo

    Para administrar las restricciones de reproducción, siga estos pasos:

    1. Configurar su cuenta
    2. Definir restricciones
    3. Asociar restricciones a un vídeo

    4. Opcional: Definir restricciones a nivel de usuario

    5. Configurar su reproductor

    Configurar su cuenta

    Esta función está disponible para un conjunto específico de clientes con acceso a la fase de disponibilidad limitada del Servicio de administración de derechos de reproducción. Consulte los detalles al gestor de su cuenta.

    Definir restricciones

    Utilice la API de derechos de reproducción para definir restricciones de reproducción de vídeo.

    API de derechos de reproducción

    Cada objeto de restricción de derechos de reproducción se puede utilizar con uno o más vídeos.

    URL base

    La URL base para la API es: 

    https://playback-rights.api.brightcove.com/v1

    Ruta de la cuenta

    En todos los casos, se solicitarán una cuenta de Video Cloud específica. Por lo tanto, siempre agregará las cuentas de término seguidas de su ID de cuenta a la URL base: 

    https://playback-rights.api.brightcove.com/v1/accounts/{accountID}

    Autorización

    Se requiere un token de acceso para solicitudes y debe estar presente en el encabezado Autorización። 

    Authorization: Bearer {access_token}

    El token de acceso es un token de acceso temporal de OAuth2 que debe obtenerse del servicio OAuth de Brightcove. Para obtener información detallada sobre cómo obtener credenciales de cliente y usarlas para recuperar tokens de acceso, consulte la descripción general de OAuth de Brightcove.

    Permisos

    Las solicitudes a la API de derechos de reproducción deben realizarse desde credenciales de cliente con los siguientes permisos:

    • video-cloud/playback-auth/playback-rights/read
    • video-cloud/playback-auth/playback-rights/write

    Administrar restricciones

    La API de derechos de reproducción admite las siguientes solicitudes. Para obtener más información sobre la API, consulte la Referencia de API de derechos de reproducción.

    Crear nuevos derechos de reproducción

    POST /v1/accounts/{accountID}/playback_rights
  Content-Type: application/json
  Body: {playback rights object}

    Para obtener una lista de campos válidos, consulte la sección Cuerpo de la solicitud .

    Ejemplo de Curl

    Crear derecho de reproducción válido sólo para Australia. 

    curl -X POST \
  https://playback-rights.api.brightcove.com/v1/accounts/{your_account_id}/playback_rights\
  -H 'Autorización: Portador {access_token} '\
  -H 'Tipo de contenido: aplicación/json'\
  -d '{
  «geo»: {
    «allowed_countries»: [
      «AU»
    ]
  }
} '

    Respuesta de API

    Guarde el playback_rights_id valor para su uso posterior. Puede encontrar este valor en la respuesta de la API. O bien:

    • Encabezado de respuesta:

      El playback_rights_id valor se puede encontrar en el Location campo de la respuesta de encabezado. Debería ser similar a esto: 

      Location: /v1/accounts/your account_id/playback_rights/your playback_rights_id

    • Cuerpo de respuesta

      El playback_rights_id valor se puede encontrar en el cuerpo de respuesta como id campo. Debería ser similar a esto: 

      {
  "id": "your playback_rights_id",
  "geo": {
  "allowed_countries": [
    "MX",
    "US"]
}

    Obtener todos los derechos de reproducción de una cuenta

    GET /v1/accounts/{accountID}/playback_rights

    Obtener una reproducción específica para una cuenta

    GET /v1/accounts/{accountID}/playback_rights/{playbackRightsID}

    Actualizar un derecho de reproducción específico:

    PUT /v1/accounts/{accountID}/playback_rights/{playbackRightsID}
  Content-Type: application/json
  Body: {playback rights object}

    Para obtener una lista de campos válidos, consulte la sección Cuerpo de la solicitud .

    Eliminar un derecho de reproducción específico:

    DELETE /v1/accounts/{accountID}/playback_rights/{playbackRightsID}

    Aquí están los detalles del campo:

    Campo Tipo Descripción
    allowed_countries, blocked_countries Cadena Arreglo de códigos de país de dos letras, que siguen la norma ISO 3166-1 alpha-2. Para obtener una lista de valores, consulte los elementos de código asignados oficialmente.
    allowed_zip_codes, blocked_zip_codes Cadena Matriz de códigos postales, que tienen el prefijo del país de dos letras y el guión ["US-90045"].
    El código de país de dos letras debe ser mayúscula y seguir la norma ISO 3166-1 alpha-2, como se muestra en los elementos de código asignados oficialmente.
    allowed_dmas, blocked_dmas Entero Matriz de números de área de mercado designada por Nielsen (DMA). Para obtener una lista de valores, consulte el documento Códigos DMA .
    blocked_proxies: anonymous Booleano La dirección IP del cliente no está disponible. Incluye servicios que cambian de ubicación para superar DRM, puntos TOR, proxies temporales y otros servicios de enmascaramiento.
    blocked_proxies: public Booleano Múltiples usuarios desde una ubicación que permite el acceso público a Internet.
    blocked_proxies: corporate Booleano Generalmente se considera inofensivo, pero la ubicación puede ser una preocupación. Identifique si varios usuarios tienen proxy a través de una ubicación central o ubicaciones, y puede compartir una única dirección IP aparente de red.
    blocked_proxies: transparent Booleano La dirección IP del cliente está disponible a través de encabezados HTTP, aunque el valor no es necesariamente confiable (por ejemplo, puede ser falsificado).
    allowed_domains, blocked_domains Cadena Matriz de nombres de dominio
    start_time, end_time Entero Tiempo de la Época
    allowed_ips, blocked_ips Entero Matriz de direcciones ipv4/ipv6 o bloques CIDR.
    allowed_days Cadena Matriz de nombres en minúsculas de 3 letras para los días en que se permite recuperar el recurso. Uno o más de: mon, tue, wed, thu, fri, sat, sun
    allowed_hours Entero Matriz de horas desde el reloj de 24 horas (comenzando en 0 y hasta 47) durante las cuales se permite recuperar el recurso. 0 a 23 para el día actual y de 24 a 47 para la fecha de finalización del día siguiente. Si un bloque de horas permitidas comienza en el día anterior y termina en el día siguiente, entonces se requiere una notación de 24+.

    Ejemplo: el valor de 3-4 en este encabezado significa que el recurso está disponible de 3:00am UTC a 3:59am UTC
    allow_insecure Booleano Preajuste: false
    Estableciendo esto en true hace que el token JWT sea opcional.
    disabled Booleano Preajuste: false
    Establece esta opción para true deshabilita la Reproducción derecha, permitiendo la reproducción para todos.
    name Cadena Nombre del derecho de reproducción opcional

    Asociar restricciones a un vídeo

    Utilice el CMS API para asociar un identificador de derechos de reproducción con un vídeo. Usará el ID de derechos de reproducción que creó en el paso anterior.

    CMS API

    Cada objeto de restricción de derechos de reproducción se puede utilizar con uno o más vídeos.

    URL base

    La URL base para la API es: 

    https://cms.api.brightcove.com/v1

    Ruta de la cuenta

    En todos los casos, se solicitarán una cuenta de Video Cloud específica. Por lo tanto, siempre agregará las cuentas de término seguidas de su ID de cuenta a la URL base: 

    https://cms.api.brightcove.com/v1/accounts/{accountId}

    Autorización

    Se requiere un token de acceso para solicitudes y debe estar presente en el encabezado Autorización። 

    Authorization: Bearer {access_token}

    El token de acceso es un token de acceso temporal de OAuth2 que debe obtenerse del servicio OAuth de Brightcove. Para obtener información detallada sobre cómo obtener credenciales de cliente y usarlas para recuperar tokens de acceso, consulte la descripción general de OAuth de Brightcove.

    Permisos

    Las solicitudes a la API de derechos de reproducción deben realizarse desde credenciales de cliente con los siguientes permisos:

    • video-cloud/video/read
    • video-cloud/video/update

    Administrar restricciones

    El CMS API admite los muchos tipos de solicitud. Para actualizar un vídeo, utilice lo siguiente:

    Actualizar un vídeo:

    Asociar un playback_rights_id con un vídeo. Este id debe existir en la API de derechos de reproducción, que creó en el paso anterior. 

    PATCH /v1/accounts/{account_id}/videos/{video_id}
  Content-Type: application/json
  Body: {video object}
    Ejemplo de Curl

    Agregar un playback_rights_id a un vídeo. 

    curl -X PATCH\
  https://cms.api.brightcove.com/v1/accounts/su account_id/videos/su video_id\
  -H 'Autorización: Portador su access_token' \
  -H 'Tipo de contenido: aplicación/json'\
  -d '{
	«playback_rights_id»: "su playback_rights_id»
} '

    Obtén un vídeo específico:

    GET /v1/accounts/{account_id}/videos/{video_ids}

    Para obtener detalles completos sobre el uso de la API, consulte la Referencia de la API de CMS.

    Definir restricciones a nivel de usuario

    Si desea restricciones a nivel de usuario, necesitará un JSON Web Token (JWT) configurado con reclamaciones asociadas a sus restricciones.

    Pasos

    Para crear un JSON Web Token (JWT) para pasar a su jugador, siga estos pasos:

    1. Determinar las restricciones

      Determine el tipo de restricciones de nivel de usuario que desea utilizar en la lista siguiente. Incluya las reclamaciones JWT necesarias cuando cree su JSON Web Token (JWT).

      Límites de dispositivos

      Cuando se realiza una solicitud de licencia DRM, se registra el dispositivo del usuario y se asigna un identificador único. El límite de dispositivos se comprueba y se aplica con cada solicitud de licencia. Para utilizar esta restricción, genere un JWT con el ID de usuario actual (uid).

      Para obtener más información, consulte el documento Implementación de límites de dispositivos .

      Límites de flujo simultáneos

      Cuando se realiza una solicitud de licencia DRM, se registra el dispositivo del usuario y se asigna un identificador único. Con cada solicitud de licencia, se comprueba el ID del dispositivo para asegurarse de que solo está viendo un número especificado de secuencias de vídeo al mismo tiempo.

      Para obtener más información, consulte el documento Limitar flujos simultáneos por visor .

      Etiquetas

      Basándose en la tags reclamación, el JSON Web Token (JWT) especificará el grupo de vídeos que el usuario tiene derecho a ver. Puede agrupar vídeos con etiquetas mediante el módulo multimedia de Video Cloud Studio de Brightcove.

      El Servicio de autorización de reproducción comprobará las etiquetas asociadas a cada vídeo. Si al menos una de las etiquetas coincide con la lista en el token JWT, entonces el video es visible.

      Las etiquetas en el token JWT se mostrarán como una matriz de etiquetas.

      Está limitado a una matriz de 40 etiquetas en el token JWT.

    2. Crear un JSON Web Token

      Para crear un token que se utilizará al comunicarse con la API de reproducción de Brightcove, consulte el documento Creating a (JWT).JSON Web Token
